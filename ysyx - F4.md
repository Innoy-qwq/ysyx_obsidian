#计算机理论
今天我们来尝试学习计算机系统的状态机模型。我们如今得以结合数电的形式来对指令进行理解。

---
### 基本概念

首先我们获取以下两个概念：

操作码 `opcode` -> 指定数据方向
操作数 `operand`  -> 指定数据位置

```
+--------+---------------+
| opcode |    operand    |
+--------+---------------+
```

原寄存器 用于操作码取值的寄存器

>`rs1`、`rs2`表示原寄存器，`R[rs1]`,`R[rs2]`表示寄存器表示的值.
>当取值并不从原寄存器取值，而是直接从指令中的二进制获取，这种数被称为`立即数`
>rs表示`source register`, rs表示`destination register`

目的寄存器 用于操作码存值的寄存器

>目的寄存器一般记为`rd`

一下给出了一些指令的使用案例，其中指令都被编码为二进制码。`add`为`00`, `li`为`10`
即`操作码+操作数`或`操作码+立即数`的组织形式，

```
00100001   add指令, 将R[0]和R[1]相加, 结果写入R[2]
10110000   li指令, 将立即数0000写入R[3]
10000101   li指令, 将立即数0101写入R[0]
```

### 计算机的相比之计算器的内核

计算机超越计算器的核心在于，指令可以以编码的形式存储于外部的存储器（比如内存），机器可以自动化调度指令的运行。而我们需要一样东西来指示计算机应当取哪一条指令，这被称之为**程序计数器**。
拥有程序计数器，我们的机器得以通过以下形式来执行指令：

```
重复以下步骤:
  从PC指示的存储器位置取出指令
  执行指令
  更新PC
```

当然我们的计算机通过指令存储器，可以以非顺序的形式执行指令. 这与计数器的顺序形式当然是冲突的，因此也许我们需要新的指令来指示指令位置。现在我们学习一个新指令：

```
 7  6 5         2 1  0
+----+---- -----+-----+
| 11 |   addr   | rs2 | if (R[0]!=R[rs2]) PC=addr bner0指令, 若不等于R[0]则跳转
+----+----------+-----+
```

这条`bner0`指令全称为`Branch If Not Equal r0`,  它判断`R[rs2]`是否等同于`R[0]`，若不相等则将PC寄存器更新为`addr`,即让PC指向`addr`处的指令。

此处的`bner0`指令自动将`R[rs2]`与`R[0]`进行比较，即将`rs2`的值与地址`0`处的值进行比较。由于地址`0`是硬件接到0的线路, 因此取值必然为0。指令的含义进一步延申即: 

```
判断`rs2`值是否为0, 非0则跳转至`addr`
```

>PC寄存器并不用于处理一般数据，因此不属于通用寄存器`GPR`.

### 典型案例：数列求和

假设我们有四个通用寄存器: `r0`, `r1`, `r2`, `r3`, 以及额外的`PC`
我们暂时不使用操作码的形式，以字母指令的形式先观察以下指令, 它实现的功能是0~10的数列求和: 

```
0: li r0, 10   # 这里是十进制的10
1: li r1, 0
2: li r2, 0
3: li r3, 1
4: add r1, r1, r3
5: add r2, r2, r1
6: bner0 r1, 4
7: bner0 r3, 7
```

看起来让人一头雾水，我们只能机械式的根据指令内容，逐步人工执行：

```
PC r0 r1 r2 r3
(0, 0, 0, 0, 0)   # 初始状态
(1, 10, 0, 0, 0)  # 执行PC为0的指令后, r0更新为10, PC更新为下一条指令的位置
(2, 10, 0, 0, 0)  # 执行PC为1的指令后, r1更新为0, PC更新为下一条指令的位置
(3, 10, 0, 0, 0)  # 执行PC为2的指令后, r2更新为0, PC更新为下一条指令的位置
(4, 10, 0, 0, 1)  # 执行PC为3的指令后, r3更新为1, PC更新为下一条指令的位置
(5, 10, 1, 0, 1)  # 执行PC为4的指令后, r1更新为r1+r3, PC更新为下一条指令的位置
(6, 10, 1, 1, 1)  # 执行PC为5的指令后, r2更新为r2+r1, PC更新为下一条指令的位置
(4, 10, 1, 1, 1)  # 执行PC为6的指令后, 因r1不等于r0, 故PC更新为4
(5, 10, 2, 1, 1)  # 执行PC为4的指令后, r1更新为r1+r3, PC更新为下一条指令的位置
(6, 10, 2, 3, 1)  # 执行PC为5的指令后, r2更新为r2+r1, PC更新为下一条指令的位置
(4, 10, 2, 3, 1)
(5, 10, 3, 3, 1)
(6, 10, 3, 6, 1)
(4, 10, 3, 6, 1)
(5, 10, 4, 6, 1)
...
(4, 10, 9, 46, 1)
(5, 10, 10, 46, 1)
(6, 10, 10, 55, 1)
(7, 10, 10, 55, 1) # 相加结束, 进入死循环
(7, 10, 10, 55, 1)
(7, 10, 10, 55, 1)
...
```


计算到一半，这段指令的作用方式已经昭然若揭：`r0`作为终止flag，`r1`作为计数器，`r2`作为缓存，`r3`作为累加步长。这样以来可以实现一个计算循环。最终当计数器累积到10, 累加的最终结果保存在了`r2`。

>其实现方式让我们想起高级语言的循环语法，参数同样是起始、终止、与步长。尤其是c语言。

我们知道，上述的表达方式其实就是汇编，而当我们转化为机器语言，也就是经过操作码和操作数的编码，结果更令人震撼: 

```
10001010    # 0: li r0, 10
10010000    # 1: li r1, 0
10100000    # 2: li r2, 0
10110001    # 3: li r3, 1
00010111    # 4: add r1, r1, r3
00101001    # 5: add r2, r2, r1
11010001    # 6: bner0 r1, 4
11011111    # 7: bner0 r3, 7
```

就是这样的一串一串二进制数，实现了循环计算，它们同时也是计算机内核。

接下来我们可以完成作业了，题目是实现一个奇数相加，理解了上述的代码，相信很容易就能完成这一项工作: 

```
0: li r0, 9 
1: li ri, 1

3: li r3, 2
4: add r1, r1, r3

7: bner0 r3, 7
```

### 指令集架构的状态机模型

指令集架构: Instruction Set Architecture , 即 ISA。要理解ISA究竟是什么，我们也许通过状态机模型可以很好地达到这一目的。状态机的单元如下: 

- 状态集合 `S{S1, S2 ...}`
- 激励事件集合 `E`
- 状态转移规则 `next`
- 初始状态 `S0∈S`

对于指令集而言，状态集合即`PC, r0, r1 ...` 一切具有可变性质的事物, 激励事件即运行指令，状态转移规则即指令的规则作用，初始状态则是混沌的。

>类似于控制论与系统论，状态机模型是一套理解事物的角度，完全可以利用此理论来理解更多事物，而指令集架构不过是在这一角度下可以很好地展开呈现而已。在过去，我们也许早在在各处看到状态机模型的存在，比如在神经网络、优化算法等领域。

### 状态机模型的拓展

如下一段代码，此刻的你已经再熟悉不过

```c
#include <stdio.h>

/* This is
   an example. */
int main() {
  int x = 1;
  int y = 2;
  int z = x + y;
  printf("z = %d\n", z);
  return 0;
}
```

但我们从状态机模型的角度去考虑，可以有新的理解：

对于这一段代码而言，状态集合S也就是各个变量以及指示程序运行到哪一步的程序计数器 (PC)， 而激励事件是程序的运行，状态转移规则是每一步程序的指令效果，而初始状态，我们知道这与编译器有一些联系，有的编译器会自动帮你变量清零，但就更原始的角度而言，变量的初始状态其实是混乱的，而PC则默认为0。

类似的，我们再用到数字电路上，实际上在写汇编的时候，很难不联想到数字电路实际在发生什么，因此也极容易理解：

状态集合S：时序逻辑元件的存储值
初始状态S0：数字电路触发器的复位状态
激励事件集合：外部的激励信号集合
状态转移规则：组合逻辑电路的结构

实验要求使用状态机角度考虑数列求和加法器的状态变化过程，这里给出分析: 

```
   rs1       rs2     
(00000000, 00000000) # 初始状态
(00000001, 00000001) # 状态转移规则；rs1 = rs1 + 1, rs2 = rs2 + rs 1
(00000010, 00000011)
(00000011, 00000110)
(00000100, 00001010)
(00000101, 00001111)
(00000110, 00010101)
(00000111, 00011100)
(00001000, 00100100)
(00001001, 00101101)
(00001010, 00110111) # 得到55
```

>在数电中，我们学习过时序逻辑电路的状态转移方程和状态转移图，使用的就是状态机的思想。事实上当我们刚开始接触时序逻辑电路，列出其状态表的时候已经开始接触状态机的思想了。

接下来我们再回顾前面的灵感，c语言的循环与汇编中的写法相当类似。虽然我第一时刻想到的是for循环，毕竟其建立初值，建立比对条件，建立步长的思路和前面的汇编代码出奇之类似，但这里给出的do while循环与汇编循环也有很大的相似性。从状态机的角度我们开始逐帧分析：

```c
#include <stdio.h>

/* 1 */ int main() { 
/* 2 */   int sum = 0; //给予变量初值，类似于li指令
/* 3 */   int i = 1; //给予变量初值，类似于li指令
/* 4 */   do {
/* 5 */     sum = sum + i; //相加，类似于ADD指令
/* 6 */     i = i + 1; //相加，类似于ADD指令
/* 7 */   } while (i <= 10); //进行数值比对和指令跳转，可以想到bner0指令
/* 8 */   printf("sum = %d\n", sum); //数值输出，其实也就是修改外部某个寄存器
/* 9 */   return 0;
/* 10*/ }
```

同时我们对比汇编版本

```assembly

        .string "sum = %d\n"
main:

        mov     rbp, rsp

.L2:
        mov     eax, DWORD PTR [rbp-8]

        cmp     DWORD PTR [rbp-8], 10
        jle     .L2
        mov     eax, DWORD PTR [rbp-4]
        mov     esi, eax
        mov     edi, OFFSET FLAT:.LC0
        mov     eax, 0
        call    printf
        mov     eax, 0
        leave
        ret
```

可以看到一些对应的代码。

### 从状态机的角度来看，CPU设计是什么？

|        | ISA          | 数字电路      |
| ------ | ------------ | --------- |
| 状态     | `{PC, R, M}` | 时序逻辑电路    |
| 激励事件   | 执行指令         | 处理组合逻辑    |
| 状态转移规则 | 指令的语义        | 组合逻辑电路的逻辑 |

CPU设计即：用数字电路的状态机去实现ISA的状态机
