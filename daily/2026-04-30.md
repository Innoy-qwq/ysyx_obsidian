### 今日解决疑问:

为什么c中%运算符的结果总是与被除数同号

>c的`(a/b)*b+a%b`的值总是等于a，换算一下即 `a%b=a-(a/b)*b`，而`a/b*b`的结果肯定是小于等于a的。

以下代码中，为什么第一个print 4，第2个print 0？
```c
double myround(double x){
	if (x>=0) return ceil(x);
	else return floor(x);
}

int main(){
	int x = myround(3.14);
	printf("%d\n", x);
	printf("%d\n", myround(3.14));
} 
```

>当进行x初始化时，c进行了显式类型强制转换，double换成int，后续%d是支持整数的。
>没经过x初始化的myround(3.14)返回double类型，而%d占位符并不支持。

为什么在printf中，%f和%lf是等价的?

> printf属于**变参函数**，变参函数会发生默认参数提升，所以就算给予单精度浮点数，函数内也会被提升为双精度，所以%f其实是支持双精度的，%lf和%f是一样用的。
> 
> scanf与printf原理就完全不同。scanf虽然也是变参函数，但给的是参数地址，所以参数没什么好提升的。而读取地址并以给出的占位符类型读数，这里单精度和双精度是有区别的。
