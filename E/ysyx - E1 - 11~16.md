## 第11章：排序与查找

插入排序和时间复杂度的概念就不复述了，其实时间复杂度的意义很多人还不是很明白，但如果做过数学建模，搞优化算法之类，很多时候在跑你的算法之前就要先估量一下复杂度，否则可能等一辈子都等不到。人对大数字的概念是抽象的，算法的可行性有的时候只需要简单估量复杂度就能知道，但也有的时候要算出来运行时长才能确切清楚，甚至有的时候要跑一下样本才行。

### 归并排序

算法即分组后先排序再合并的思路。从树的角度考虑它的复杂度其实非常简单，归并排序每一层深度都要n的复杂度，然后深度是$\log_2n$, 所以整体的复杂度也就是$n\log_2n$。

#### 习题：

1、快速排序是另外一种采用分而治之策略的排序算法，在平均情况下的时间复杂度也是Θ(nlgn)，但比归并排序有更小的时间常数。请参考以下代码补完`partition`函数。

```c
int partition(int start, int end)
{
	从a[start..end]中选取一个pivot元素（比如选a[start]为pivot）;
	在一个循环中移动a[start..end]的数据，将a[start..end]分成两半，
	使a[start..mid-1]比pivot元素小，a[mid+1..end]比pivot元素大，而a[mid]就是pivot元素;
	return mid;
}

void quicksort(int start, int end)
{
	int mid;
	if (end > start) {
		mid = partition(start, end);
		quicksort(start, mid-1);
		quicksort(mid+1, end);
	}
}
```
