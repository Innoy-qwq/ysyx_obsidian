## 第11章：排序与查找

插入排序和时间复杂度的概念就不复述了，其实时间复杂度的意义很多人还不是很明白，但如果做过数学建模，搞优化算法之类，很多时候在跑你的算法之前就要先估量一下复杂度，否则可能等一辈子都等不到。人对大数字的概念是抽象的，算法的可行性有的时候只需要简单估量复杂度就能知道，但也有的时候要算出来运行时长才能确切清楚，甚至有的时候要跑一下样本才行。

### 归并排序

算法即分组后先排序再合并的思路。从树的角度考虑它的复杂度其实非常简单，归并排序每一层深度都要n的复杂度，然后深度是$\log_2n$, 所以整体的复杂度也就是$n\log_2n$。

#### 习题：

1、快速排序是另外一种采用分而治之策略的排序算法，在平均情况下的时间复杂度也是Θ(nlgn)，但比归并排序有更小的时间常数。请参考以下代码补完`partition`函数。

```c
int partition(int start, int end)
{
	从a[start..end]中选取一个pivot元素（比如选a[start]为pivot）;
	在一个循环中移动a[start..end]的数据，将a[start..end]分成两半，
	使a[start..mid-1]比pivot元素小，a[mid+1..end]比pivot元素大，而a[mid]就是pivot元素;
	return mid;
}

void quicksort(int start, int end)
{
	int mid;
	if (end > start) {
		mid = partition(start, end);
		quicksort(start, mid-1);
		quicksort(mid+1, end);
	}
}
```

>代码如下：

```c
int a[] = {3, 6, 8, 10, 1, 2, 1};

void swap(int i, int j)
{
	int temp = a[i];
	a[i] = a[j];
	a[j] = temp;
}
int partition(int start, int end)
{
	int i = start, j = start - 1;
	int pivot = end; 

	// i作为遍历，j作为比a[pivot]大或者小的分界, pivot指代要选取的值的位置
	while (i <= end){
		// 遇到比pivot小的数，就把数扔到左边，然后j往前推进一格。j保证了一个锁定功能,确保了j自身以及前面的绝对是小于a[pivot]的
		if (a[i] < a[pivot]){
			j++;
			swap(i, j);
		} 
		i++;
	}
	swap(pivot, j+1);
	return j+1;
}

void quicksort(int start, int end)
{
	int mid;
	if (end > start) {
		mid = partition(start, end);
		quicksort(start, mid-1);
		quicksort(mid+1, end);
	}
}

int main()
{
    int n = sizeof(a) / sizeof(a[0]);
    quicksort(0, n-1);
	for (int i = 0; i < n; i++){
		printf("%d ", a[i]);
	}
    return 0;
}
```

3、进一步泛化，在一组随机排列的数中找出第k小的，这个元素称为k-th Order Statistic。能想到的最直观的算法肯定是先把这些数排序然后取第k个，时间复杂度和排序算法相同，可以是Θ(nlgn)。这个问题虽然比前两个问题复杂，但它也有平均情况下时间复杂度是Θ(n)的算法，将上一节习题1的快速排序算法稍加修改就可以解决这个问题：

```c
/* 从start到end之间找出第k小的元素 */
int order_statistic(int start, int end, int k)
{
	用partition函数把序列分成两半，中间的pivot元素是序列中的第i个;
	if (k == i)
		返回找到的元素;
	else if (k > i)
		从后半部分找出第k-i小的元素并返回;
	else
		从前半部分找出第k小的元素并返回;
}
```
请编程实现这个算法。

>答：代码如下：

```c
int a[] = {3, 6, 8, 10, 1, 2, 1};

void swap(int i, int j)
{
	int temp = a[i];
	a[i] = a[j];
	a[j] = temp;
}
int partition(int start, int end)
{
	int i = start, j = start - 1;
	int pivot = end; 

	// i作为遍历，j作为比a[pivot]大或者小的分界, pivot指代要选取的值的位置
	while (i <= end){
		// 遇到比pivot小的数，就把数扔到左边，然后j往前推进一格，其实j也保证了一个锁定功能,确保了j自身以及前面的绝对是小于a[pivot]的
		if (a[i] < a[pivot]){
			j++;
			swap(i, j);
		} 
		i++;
	}
	swap(pivot, j+1);
	return j+1;
}

int order_statistic(int start, int end, int k)
{
	k++; //k+1来对应数组索引
	int i = partition(start, end);
	if (k == i)
		return a[i];
	else if (k > i)
		order_statistic(i+1, end, k-i);
	else
		order_statistic(start, i-1, k);
}

int main()
{
    int n = sizeof(a) / sizeof(a[0]);
    int ans = order_statistic(0, n, 3);
	printf("%d ", ans);
    return 0;
}
```

### 折半查找

#### 习题：

1、本节的折半查找算法有一个特点：如果待查找的元素在数组中有多个则返回其中任意一个，以本节定义的数组`int a[8] = { 1, 2, 2, 2, 5, 6, 8, 9 };`为例，如果调用`binarysearch(2)`则返回3，即`a[3]`，而有些场合下要求这样的查找返回`a[1]`，也就是说，如果待查找的元素在数组中有多个则返回第一个。请修改折半查找算法实现这一特性。

 >答：代码如下：其实只加了一行while循环

```c
#define LEN 8
int a[LEN] = { 1, 2, 2, 2, 5, 6, 8, 9 };

int binarysearch(int number)
{
	int mid, start = 0, end = LEN - 1;

	while (start <= end) {
		mid = (start + end) / 2;
		if (a[mid] < number)
			start = mid + 1;
		else if (a[mid] > number)
			end = mid - 1;
		else
			while (number == a[mid]) mid--;	
			return mid+1;
	}
	return -1; //-1表示没有找到
}

int main(void)
{
	printf("%d\n", binarysearch(2));
	return 0;
}

```

2、编写一个函数`double mysqrt(double y);`求`y`的正平方根，参数`y`是正实数。我们用折半查找来找这个平方根，在从0到`y`之间必定有一个取值是`y`的平方根，如果我们查找的数`x`比`y`的平方根小，则$x^2<y$，如果我们查找的数`x`比`y`的平方根大，则$x^2>y$，我们可以据此缩小查找范围，当我们查找的数足够准确时（比如满足$|x^2-y|<0.001$），就可以认为找到了`y`的平方根。思考一下这个算法需要迭代多少次？迭代次数的多少由什么因素决定？

>答：迭代次数取决于精度要求和数值大小。因为实际上这相当于在一个步长为精度值的数组中进行二分法，假设精度要求为$k$, 那么数组大小$n = y/k$, 而二分法得到的复杂度也就是$log_2(y/k)$。

3、编写一个函数`double mypow(double x, int n);`求`x`的`n`次方，参数`n`是正整数。最简单的算法是：

```c
double product = 1;
for (i = 0; i < n; i++)
	product *= x;
```

这个算法的时间复杂度是Θ(n)。其实有更好的办法，比如`mypow(x, 8)`，第一次循环算出x·x=x2，第二次循环算出x2·x2=x4，第三次循环算出4·x4=x8。这样只需要三次循环，时间复杂度是Θ(lgn)。思考一下如果`n`不是2的整数次幂应该怎么处理。请分别用递归和循环实现这个算法。

>答：代码如下：以后的想法内容直接写在注释了，这样比较方便。

```c
// 求x的n次方
// 当n不是2的整数次幂时，我们可以将其拆解为多个2的整数次幂的相加。
// 这样比如求x的6次方，9 = 1 + 8, 即2的1次方和3次方相加。result可以裂解为 x * [(x*x * x*x) * x*x]
// 这里要拆解为2的整数次幂的根本原因在于，乘法运算是2元运算，你要保证每一次2元都是你手头里最大的数，因此拆解成其他的整数次幂相加必然比2更差一些.
// 具体而言，比如也许你会想第一步先算x的三次方再复用，这也许能在后续每次数值大一些，但多算的这一步已经弥补不回来了。写一个比对就能看出来：
// 2 4 8 16 32 64
// 2 3 6 12 24 48
// 那么怎么把n分解为2的整数次幂呢？最好的方法莫过于直接读其二进制了。比如9是1001，这里我们先利用一下位运算：
double mypow(double x, int n){
	double result = 1;
	while (n > 0){ //如果是负数防死循环
		if ( n&1 ) result *= x; // 判断对应位是不是0，如果是0则跳过，是1则乘上去。
		x *= x; // 每一次升高位，x都变为x的对应位次方，这里x其实复合了计数和计算的责任
		n >>= 1; //移位
	}
	return result;
}
int main(void)
{
	printf("%f\n", mypow(2.8, 3));
	return 0;
}

```

```c
// 递归版本,至少传3个量，负责计数、算子a，算子b
double mypow(double x, int n, double result)
{

	if ( n&1 ) result *= x; // 判断对应位是不是0，如果是0则跳过，是1则乘上去。
	x *= x; // 每一次升高位，x都变为x的对应位次方
	n >>= 1; //移位

	if (!n) return result;
	mypow(x, n, result);
}

int main(void)

}
```

## 第12章：栈与队列

数据结构的构建和使用我在数学建模已经品鉴过太多次了。针对一个问题，设计出好的数据结构对对象进行整理是相当相当重要的，这也就是所谓数据的“形状”，是几何方法思考的基础。《动手学深度学习》的作者李沐曾说，搞深度学习很多时候也不需要怎么调试，只要形状对了，结果大概也就是对的。为什么数据结构能有这样的验证作用？恰恰就是因为当你对问题理解的清楚，自己设计的数据结构含义就是十分清楚的，那么每个模块的张量形状只要对上号，结果意义也一定是明确的。此外，即使自己理解的有问题，结果错了，那这个结果也应当具备某种其他含义，因为这个过程并没有破坏熵值，低熵的结果肯定是具有某种含义的。

就我个人体感而言，面对一个建模问题有一个大概算法方向后，首先就是要厘清对象的数据结构，这是编程手的首要目标工作，这同时也考验你对问题本身的理解水平。理清数据结构之后，针对算法就要设计出多个方便自己调度计算的数据结构，这往往是过程性的、变化的一个过程，不可能一个数据结构解决所有问题：很多时候问题分解之后，针对子问题又要组织一次数据结构。

在工程中，数据结构的设计同时也是安全性的保证。如果一个地方先前被设计成栈，那就说明原编写者在考虑这个问题时，这里就不应当承担有其他特性。如果自己要额外变动，给这个数据结构加特性，那就一定要谨慎做好沟通工作，理解双方对问题的看法。

虽然说了这么多，但是我们接下来要学的依然是两种最经典的数据结构，栈和队列。他们之所以经典，也是因为它们确实是一个非常通用的可泛化模型，能套用到很多问题上去。

### 堆栈与深度优先搜索

这里先不谈数据结构，物理意义上的堆 (heap) 与栈 (stack) 的区别是老生常谈的话题，但很可惜我忘了（）印象里是堆储存在全局作用域，栈存储于函数作用域？再复习一下。

堆：`malloc`/`new`来人工创建，所以堆也就是通常意义上我们我们常说的“开一块内存”的内存，它的建立方式与操作系统紧密相关，可能是虚拟内存，可能是碎片的也可能是紧密的，甚至不够用的时候可以向操作系统内核申请。

栈：我们知道在RISC-V中，ISA层面就已经规定了`x2`作为栈指针存放的专用寄存器`sp`。因此栈与硬件关联是比较强的，抽象层次上比操作系统全程管理的堆低一些。栈的操作几乎全部由**cpu指令**直接驱动，只有操作越界和初始化线程时OS才会参与，这也解释了为什么栈的速度更快。而且代码的栈申请是在编译阶段就定死的，整体是线性的和连续的。

好吧，复习完物理意义的栈和堆，现在就需要搞清楚一件事：本章节谈到的数据结构的栈，只是作为一种数据组织的形式，和上述的栈是两个东西。

深度优先搜索的走迷宫算法就不提了，其实这个拿递归就很方便，或者说递归本身就是一个函数栈的使用，两者是一体的，因此用到递归的地方都可以拿栈来栈替换。
