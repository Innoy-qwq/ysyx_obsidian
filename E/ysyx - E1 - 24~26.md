2、编一个函数，输入一个字符串，要求做一个新字符串，把其中所有的一个或多个连续的空白字符都压缩为一个空格。这里所说的空白包括空格、'\t'、'\n'、'\r'。例如原来的字符串是：

```
This Content hoho       is ok
        ok?

        file system
uttered words   ok ok      ?
end.
```
压缩了空白之后就是：

```
This Content hoho is ok ok? file system uttered words ok ok ? end.
```

实现该功能的函数接口要求符合下述规范：

```
char *shrink_space(char *dest, const char *src, size_t n);
```

各项参数和返回值的含义和`strncpy`类似。完成之后，为自己实现的函数写一个Man Page。

>答: 代码如下:

```c
#include <stdio.h>

// 函数原型应当是char *shrink_space(char *dest, const char *src, size_t n);
// 实现的效果是将src的字符串选取n个, 进行空格压缩并填入dest, 若字符串不满n个则后续填\0

char *shrink_space(char *dest, const char *src, size_t n)
{
    char *ret = dest;
    int flag = 0; // 表示是否已经出现空位;
    for (int count = 0; count < n; count++)
    {
        if (*src == ' '  ||
            *src == '\t' ||
            *src == '\n' ||
            *src == '\r'){
            if(!flag){
            *dest = ' ';
            dest++;
            flag = 1;}
            }
        else{
            *dest = *src;
            dest++;
            flag = 0;
            }

        if (*src != '\0')
            src++;
    }
    
    return ret;
}

int main(void)
{
    char a[30];
    printf("%s", shrink_space(a, "Hello \n \t Innoy", 15));
};
```

### malloc and free

malloc申请内存, 返回void* 类型指针, 

>注意: malloc申请的内存被free掉, 只是这一块地址不再归属于用户程序, 指向这块地址的指针当然仍是存在的, 这时我们最好手动把这个野指针指向NULL来保证安全.

### 传入参数与传出参数

传入参数就是作为指针参数进入函数后,读取其数据进行使用的参数. 传出函数就是作为指针参数进入函数后, 被写入数据用于外部读取的参数, value-result函数就是既作为传入读取也作为传出调用的指针.

>传出参数和result-value参数作为函数接口完全相同.


两层指针的用法应当很明白, 没什么额外的信息. 

### 回调函数

经典的回调函数, 函数指针作为参数被传入到函数中提供调用, 一般使用viod*.
C++的泛型算法（Generics Algorithm）中可以使用回调函数, 其作用是可以有一个灵活的比较函数, 外部调用者自行实现两个对象的比较函数, 然后将这个比较函数的函数指针作为回调函数传入就可以了. 这样的话可以实现比如结构体之间的比较. 

这里效果上就有点类似于cpp的运算符重载

/*尝试使用泛型的二分查找, 这里求助了AI作为参考, 返回这里返回了字符串的地址, 修改前面的cmp可以变成查找int, 返回int的地址, 总之返回的是数组中的地址位置*/
void *binary_search(
    void *base,
    size_t n,
    size_t size,
    void *key,
    int (*cmp)(const void *, const void *))
{
    char *arr = base;
    size_t left = 0, right = n;

    while (left < right)
    {
        size_t mid = (left + right) / 2;
        void *elem = arr + mid * size;

        int c = cmp(key, elem);

        if (c == 0)
            return elem;
        else if (c < 0)
            right = mid;
        else
            left = mid + 1;
    }

    return NULL;
}

> 泛型的三种:
>

### 可变参数

过去我们曾在第三章提到过可变参数，它是由`...`表示构成可变参数，用`va_list`和`va_start`、`va_arg`、`va_end`进行管理的一类函数，使用时需要引入外部库`stdarg.h`.

原教材并没有给出这几个类型的含义, 说实话我不理解为什么要这样做. 也许是想要让我们自行使用man来自行理解. 所以这里我贴出 `man va_start`的结果, 并给出我的阅读过程: 

首先是函数原型部分, 这一部分用于后文的参照

```

NAME
       stdarg, va_start, va_arg, va_end, va_copy - variable argument lists

LIBRARY
       Standard C library (libc, -lc)

SYNOPSIS
       #include <stdarg.h>

这里可以看到 `va_start`, `va_arg`, `va_end` 都围绕着`va_list`这个类型展开, 快速浏览一下 `va_list` 类型的man: 

```
va_list(3type)

NAME
       va_list - variable argument list

LIBRARY

SYNOPSIS
       #include <stdarg.h>

       macros va_start(3), va_arg(3), va_copy(3), and va_end(3) to traverse the list of arguments.

可以看到它的原型写着 `typedef /* ... */  va_list;` 经查阅这是表示具体实现并不展示, 不同的平台会给出不同实现, 所以这是一种不透明类型. 实际上在早期的编译器可以直接用`char*`类型, 也就是指针来用, 但是x86-64后情况变得复杂, 因为函数参数所在的寄存器可能被分割, 可能不在栈中, 因此会有更加复杂的实现. 

好了，接下来我们继续看那几个函数的man. description描述了函数的用法: 

```
DESCRIPTION
       A  function  may be called with a varying number of arguments of varying types.  The include file <stdarg.h> declares a type va_list and defines three macros for stepping through a list of arguments whose number and types are not known to the called function.

       The called function must declare an object of type va_list which is used by the macros va_start(), va_arg(), and va_end().
```

这里简单介绍了几个函数的来历, 用于可变参数. 

```
va_start()
		The va_start() macro initializes ap for subsequent use by va_arg() and va_end(), and must be called first.

       The argument last is the name of the last argument before the variable argument list, that is, the last argument of which the calling function knows the type.

       Because  the address of this argument may be used in the va_start() macro, it should not be declared as a register variable, or as a function or an array type.
```

`va_start`建立了后续用于`va_arg`和`va_end`的序列流程初始化, 必须被第一个被调用. last是用于定位参数序列读取起点的参数, 它的实参填写可变函数中想要开始序列读取的参数名就可以了.

由于`va_start()`会读取地址, 因此`last`不能被定义为`rigister`类型, 这是一种要求变量保存在寄存器的类型声明, 寄存器不能被读地址因此当然也就不能用在这里. 

`last`也不能被定义为函数, 这是因为函数被传递时会被退化(decay)为`void (*func)()`, 这样变成指针类型的话计算内存大小就可能会出问题. 同理`last`也不能被传递 `array` 类型, 毕竟数组被传递时也会decay为指针类型. 这样的话算size就变成了指针的大小而不是数组的大小. 可以想象往后读参数的时候可能把数组内容当成参数. 

> 实际上现代GCC已经没有这些限制, 因为现代GCC中`va_start`已经是编译器內建指令, 因此清楚ABI规则. 

```
   va_arg()
       The va_arg() macro expands to an expression that has the type and value of the next argument in the call. The argument ap is the va_list ap initialized by va_start(). Each call to va_arg() modifies ap so that the next call returns the next argument. The argument type is a type name specified so that the type of a pointer to an object that has the specified type can be obtained simply by adding a * to type.

       The first use of the va_arg() macro after that of the va_start() macro returns the argument after last. Successive invocations return the values of the remaining arguments.

       If there is no next argument, or if type is not compatible with the type of the actual next argument (as promoted according to the default argument promotions), random errors will occur.

       If ap is passed to a function that uses va_arg(ap,type), then the value of ap is undefined after the return of that function.
```
