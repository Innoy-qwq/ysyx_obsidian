## 第一章：C语言入门
### 编译型语言与解释型语言

我们接触过的编译型语言是c、cpp，而解释型语言则以python为典型。此外也可以有JAVA这样使用即时编译技术 (JIT, 即Just In Time) 的语言，不过这都无所谓，只要代码能够跑起来，过程怎样也都是可以的，那么编译和解释究竟是如何让代码能够跑起来的？

编译，即将高级语言转化为低级语言，这是容易想象的事情：汇编代码的指令功能对我们而言还是太陌生了，如何将陌生转化为容易想象的事情是提高编程效率的关键，而c语言的诞生就是这么做的：它建立了一套符合人类习惯的体系。此后人们只要写下c代码，然后通过某种东西，也就是编译器，转化为汇编，机器码就可以跑了。

实际上面向对象的诞生，无非也是围绕这个核心：如何将编程的过程让人们更容易想象和理解。

解释型语言的执行过程是通过将每一句代码用解释器与预留的机器语言接口进行对接来完成的，因此可以直接不用编译一句一句执行，毕竟已经编译过了。而且解释型语言一般也不会太死板以至于完全的一步一步解释，否则必须要面对一个时域问题，即非因果信号如何处理。比如我调用一个后文才定义的函数怎么办？这种功能需要先概览全文才能支持。

#### 习题：解释执行的语言相比编译执行的语言有什么优缺点？

>答：
>1. 解释执行可以实现逐步执行，但语言的检查与转接都是即时的，因此速度较慢。编译型语言通过预先阅读全文，可以做到很好的代码优化。代码的检查工作在编译时已经完成，而编译出可执行文件之后是直接脱离编译器跑的机器语言，速度是很快的。
>2. 虽然理论上编译型语言也可以逐步执行，毕竟可以走一步现场编译一步。但是这样的话实际上就失去了编译型语言的性能优势，开发环境可以考虑这样做。
>3. 解释器代码文件是平台无关可以直接跑的，而编译型语言虽然也是平台无关，但可执行文件是和平台捆绑的，要在不同平台执行需要先在本地环境编译，这不够方便。

### 形式语言与自然语言

形式语言由符号`token`和结构`structure`组成，这让我们想起亚里士多德将事物分为"质料"和“形式”。这里token就充当着语言的质料，而structure则是语言的形式。

### 程序的调试

bug分为以下三类：
1. 编译时错误。连编译器都过不去的错误，同时也是最容易解决的bug。
2. 运行时错误。编译过的了但程序运行可能在某些情况下崩溃，容易想到可能是指针改写了不该指向的地址，或者内存溢出之类就会发生这类错误，不过通过debug也相对容易发现。
3. 逻辑错误和语义错误。最麻烦的类型，有时候对使用的框架了解不深，发生这一类错误大概是最难处理的，因为自己本来就不是很清楚正在发生什么。发生这一类错误意味着对于现在正在发生什么，自己其实不是完全清楚。在F6中最后遇到的其实也算是逻辑错误。

### 转义序列与转换说明

以下是一些标准的转义字符，其中比较有趣的是`\a`, 这个古老的字符已经不被多少IDE和终端支持了，但是在Windows环境下你可以在terminal通过指令`[Console]::Beep()`体验一下。

| \'  | 单引号'（Single Quote或Apostrophe） |
| --- | ----------------------------- |
| \"  | 双引号"                          |
| \?  | 问号?（Question Mark）            |
| \\  | 反斜线\（Backslash）               |
| \a  | 响铃（Alert或Bell）                |
| \b  | 退格（Backspace）                 |
| \f  | 分页符（Form Feed）                |
| \n  | 换行（Line Feed）                 |
| \r  | 回车（Carriage Return）           |
| \t  | 水平制表符（Horizontal Tab）         |
| \v  | 垂直制表符（Vertical Tab）           |

>由于格式化字符串已经接触的够多，这里就已经不再记录其内容了, 它们又被称为转换说明。但值得注意的是转换说明与转义序列的差别：
>转义序列，比如\n，是在编译时处理的，比如在可执行文件中，\n已经被处理为换行符。
>转换说明，比如%c，是在调用`printf`函数时处理的，这是代码运行时才会转换。

>总结而言，转义字符是编译时起作用，而转换说明是运行时起作用。这也是理所当然的，毕竟转义序列是固定的，但转换说明是变量。是变量意味着它可能要随着外部环境的改变而改变，比如这个变量用于保存一个外部文件数据，而文件内容还没有读取，怎么可能在编译时就搞清楚要转义成什么呢？这是非因果的。

## 第三章：简单函数

### 自定义函数

在这一部分，我关注到的有趣事实并不多，主要在以下几点：
1. 函数声明是可以不给参数起名的。这让我们想起lambda函数。
2. 函数的声明要写在使用之前是C89的要求，这在C99中被取消。
3. `main(){...}`的形式是old style c的风格，现代代码应当主动填入`void`参数和`return 0`。
4. 函数参数空着会导致编译器不检查其参数，下文将介绍的隐式声明就会使用这一用法。

### 隐式声明

当函数在使用之前没有给出函数声明，编译器会给出一个隐式声明来保证程序运行，它的函数参数数量任意且不会被检查函数类型，而返回值类型是int。甚至C89有参数提升的规则：为了满足参数类型是完整word字长，C89标准会自动将小于int类型的参数升格到int，而浮点数升级为double。

>参数升级的目的是满足完整字长，这样的好处是电路计算更快。尽管理论这样说，而实际上完整字长为什么能够计算更快，这一点只有现在完成F阶段的你才能切身体会到，不是吗？

在C89标准中，隐式声明是被允许的特性，甚至C89还有隐式int：当变量或函数返回值没有给出类型，默认为int。在C99中，隐式声明被取消，编译器需要至少给出一个警告。

### 可变参数

让我们看一看`printf`的函数原型：

```c
int printf(const char *format, ...); 
```

这里的`...`就是所谓的可变参数，在python中则是`*`。参数升级在可变参数中仍然是生效的，而可变参数的处理则需要额外的库文件`stdarg.h`。

#### 习题1：
定义一个函数`increment`，它的作用是把传进来的参数加1，代码如下。那么我们在`main`函数中调用`increment`增加变量`i`和`j`的值，这样能奏效吗？为什么？

```c
void increment(int x)
{
	x = x + 1;
}

int main(void)
{
	int i = 1, j = 2;
	increment(i); /* i now becomes 2 */
	increment(j); /* j now becomes 3 */
	return 0;
}
```

>答：无法奏效，这也是典型的形参实参问题了，实参的值传入形参，后文参与计算的都是函数域内的参数，与前文外部作用域无关了。使用返回值赋值，或者使用指针即可修改外部作用域。

#### 习题2：
如果在一个程序中调用了`printf`函数却不包含头文件，例如`int main(void) { printf("\n"); }`，编译时会报警告：`warning: incompatible implicit declaration of built-in function ‘printf’`。请分析错误原因。

>答：这个错误是隐式声明的错误。`printf`本身作为一个函数是需要被声明的，引入库文件实际上已经帮忙声明，但没有引入的话，这就是典型的''未声明先使用''，那么就会触发隐式声明以及相关警告和错误。

### 变量与作用域

这一部分有两点要注意：
1. 全局变量的初始化只能用常量表达式。这是因为全局变量的初始化发生在编译时期，因此函数和变量这种典型的运行时内容是不能参与其初始化的。
2. 函数的定义域中是可以声明其他函数的，此时这个声明的作用域也被限制在函数作用域里。而函数的定义域中，再定义函数则是不被允许的。

## 第五章：深入理解函数

### return语句

函数的返回值相当于定义一个临时变量，并用return后的语句进行初始化。
#### 习题：
1、编写一个布尔函数`int is_leap_year(int year)`，判断参数`year`是不是闰年。如果某年份能被4整除，但不能被100整除，那么这一年就是闰年，此外，能被400整除的年份也是闰年。

>答：代码如下
```c
int is_leap_year(int year){
	if ((!(year%4)&&year%100) || !(year%400))
		return 1;
	else 
		return 0;
}

int main(void){
	printf("%d\n", is_leap_year(2000));
}
```

2、编写一个函数`double myround(double x)`，输入一个小数，将它四舍五入。例如`myround(-3.51)`的值是-4.0，`myround(4.49)`的值是4.0。可以调用`math.h`中的库函数`ceil`和`floor`实现这个函数。

>答：ceil是向上取整，floor是向下取整。比较直接的代码如下
```c
double myround(double x){
	if(x>=0)
		if ((int)(x*10)%10>=5) 
			return ceil(x);
		else 
			return floor(x);
	else
		if ((int)(x*10)%10>=-5) 
			return ceil(x);
		else 
			return floor(x);
}

int main(void){
	printf("%.0f\n", myround(-3.6));
} 
```

>直觉告诉我肯定有简洁的算法，因为这里面含有一些微妙的数学关系，而且两个判断有大量重复部分，经过优化应该是能合并的。这个算法从信息的角度考虑，必须要引入5这个数字，同时小数后1位的信息在经过5相关的运算后是要丢弃的。还有ceil和floor两个运算作为运算结果。
>那么先考虑正数，可以想到要先引入关键的小数后1位，进而引入5，则想到`x*10/5%2`作为控制位。 正数有为1则ceil，为0则floor，负数就有-1则floor，为0则ceil。这里规则虽然相反了，但是x本身带有符号位，参与运算则可以考虑相消，假设算式结果为y, 结果为1则ceil，也就有表：

| x   | y   | out |
| --- | --- | --- |
| >0  | 1   | 1   |
| >0  | 0   | 0   |
| <0  | 0   | 1   |
| <0  | -1  | 0   |

>我们要做的就是将out为1合并。这是可以弄出来并化简的，得到公式`x*((x&&y)-(x&&!y))>0`，或者将x>0作为基础单元，得到`(x>0&&y)||(x<0&&!y)`，这是异或逻辑。总之有以下代码：

```c
double myround(double x){
	int y = (int)(x*10/5)%2;
	if ((x>0&&y)||(x<0&&!y)) 
		return ceil(x);
	else
		return floor(x);
}

int main(void){
	printf("%.0f\n", myround(3.4));
	printf("%.0f\n", myround(3.6));
	printf("%.0f\n", myround(-3.4));
	printf("%.0f\n", myround(-3.6));
} 
```

### 递归

#### 习题：
1、编写递归函数求两个正整数`a`和`b`的最大公约数（GCD，Greatest Common Divisor），使用Euclid算法：
1. 如果`a`除以`b`能整除，则最大公约数是`b`。
2. 否则，最大公约数等于`b`和`a%b`的最大公约数。
Euclid算法是很容易证明的，请读者自己证明一下为什么这么算就能算出最大公约数。最后，修改你的程序使之适用于所有整数，而不仅仅是正整数。

>答：代码如下
```c
int Euclid(int a,int b){
	a = abs(a);
	b = abs(b);
	if (!(a%b)) return b;
	else Euclid(b, a%b);
}

int main(void){
	int a = 24;
	int b = 15;
	printf("%d", Euclid(a, b));
}
```

2、编写递归函数求Fibonacci数列的第`n`项，这个数列是这样定义的：
fib(0)=1  
fib(1)=1  
fib(n)=fib(n-1)+fib(n-2)

>答：代码如下
```c
int Fibonacci(int n){
if (n<=1) return 1;
else return Fibonacci(n-1)+Fibonacci(n-2);
}

int main(void){
	printf("%d", Fibonacci(5));
}
```
