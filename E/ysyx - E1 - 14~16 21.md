### 浮点数

浮点数在计算机中是使用科学计数法进行计算的，科学计数法我们知道分为`符号位`, `指数`, `尾数`。在计算机中被存放到一个地址中。一个字的大小是4字节，32位，也就是单精度浮点数的大小，这三个量可能就被按照顺序在某个地址被存放。

>值得注意的是，你还记得RISC-V的指令位宽只有32，那么它是怎么存储浮点数的？我们先前使用过LUI和ADDI在寄存器拼凑大整数，但浮点数处理是另一码事。
>
>RISC-V处理浮点数遵循IEEE 754标准，比较复杂。RISC-V具备专门的浮点寄存器`f0~f31`，浮点数不能作为立即数被加载进指令，也就是说浮点数的运输依赖地址，这个地址通常是内存。也可以曲线救国：先用LUI和ADDI在整数寄存器拼凑浮点数二进制，再用`fmv.w.x f0, x10` 指令把数从整数寄存器取到浮点数寄存器，这样的话数字就会被解释为浮点数。
> 
>那么这个浮点数是怎么被存进内存的？这就需要分静态变量和局部变量了。静态变量/全局变量是在编译过程中直接定死为二进制的，其值是直接和其他指令一样直接加载进内存。局部变量要么通过LUI+ADDI组合拼凑再运输到浮点数寄存器，要么就直接查表了。

>你会发现上述浮点数被存进内存的过程中，是随着指令一起被存进去的。这不乱吗？事实上二进制文件具备文件头`ELF header`。这让人想起.png和.jpg的魔数，文件头会说明代码段，数据段与入口。

这个浮点数相加的过程我看了半天，请教Gemini指导才明白。
浮点数计算前就会把隐藏的头1位拿回来，然后进行对阶。
浮点数对阶比我想象的简单，无非就是指数位改成相等值，而它们之间指数差了多少值，指数较低的其尾数就要右移多少位。和十进制一样。
对接后，指数位不变，尾数进行普通的加减。最后再把开头的1去掉，再剪裁一下舍去末尾存不进去的值，也就结束了。

## 第15章：数据类型讲解

### 整型

默认整形的有符号和无符号是Implementation Defined的，也就是和编译器挂钩。

值得关注的是三类未定义的差别：Implementation-defined、Unspecified和Undefined。

Implementation-define: 这一情况下，虽然c语言标准并未规定，但编译器需要规定并写入文档。

Unspecified：这一情况下，c语言标准未规定，编译器规定但不必写入文档，但必须要做处理。

Undefined：这一情况下，c语言标准和编译器均未规定，甚至编译器可能都不会对此做处理。

>有趣的是，实际上除char类型外，其余整型大小均为Implementation Defined，只是它们通常遵循ILP32和LP64规范。

### 浮点型

通常，float型占32位，double占64位。注意通常int型也是32位。

### 类型转换

我们先前在隐式声明、隐式int和可变参数中接触过Integer Promotion，也就是整型升级。我们知道这是为了满足对齐加速运算。

双目运算符两侧数字的类型不同时，会进行自动参数类型转换。

>注意：整型转换范围不够时，往往截取低位（取余）。浮点数类型则是损失精度。

## 第16章：运算符详解

### 位运算

做位运算时，数字首先会进行Intager Promotion.也就是至少被提升到32位的int型。

左移相当于整体乘2，右移相当于整体除2，也都是知道的，就好像在十进制加0减0一样，是相当于整体乘10和除10

>移位的位数超过操作数的位数属于Undifined，记得吗，这是很有可能不被处理的。

>对于有符号数，右移后是否是符号位填充，即高位补1，是不一定的，这属于Implementation-defined，对于x86和RISC-V而言，统一使用的是填充符号位。

#### 习题：

1、下面两行`printf`打印的结果有何不同？请读者比较分析一下。`%x`转换说明的含义详见[第 2.9 节 “格式化I/O函数”](https://akaedu.github.io/book/ch25s02.html#stdlib.formatio)。

```c
int i = 0xcffffff3;
printf("%x\n", 0xcffffff3>>2);
printf("%x\n", i>>2);
```

>答：打印结果是`33fffffc`和`f3fffffc`。
>对于第一个打印，`0xcffffff3`的字面值超过了有符号数的上限`0x7fffffff`，因此自动被提升为无符号数，而无符号数右移高位补0。
>因此高8位 `1100 1111 -> 0011 0011`, 低8位`1111 0011 -> 1111 1100`, 形成了 `33fffffc`。
>对于第二个打印，由于`i`已经被设定为int型，因此超过最大量的字面值最高位被理解为符号位，而符号位右移高位要补符号位。
>因此高8位 `1100 1111 -> 1111 0011`, 低8位`1111 0011 -> 1111 1100`, 形成了 `f3fffffc`。

### 掩码

掩码的概念相当于一个窗函数。

#### 习题：

1、统计一个无符号整数的二进制表示中1的个数，函数原型是`int countbit(unsigned int x);`。

>代码如下：

```c
#include <stdio.h>

// 实际上我们在11章已经用过类似的位运算函数了
int countbit(unsigned int x)
{
	int count = 0;
	while(x){		
		if (x&1) count++;
		x >>= 1;
	}
	return count;
};

int main()
{
	printf("%d\n", countbit(7));
}
```

2、用位操作实现无符号整数的乘法运算，函数原型是`unsigned int multiply(unsigned int x, unsigned int y);`。例如：(11011)2×(10010)2=((11011)2<<1)+((11011)2<<4)。

>代码如下：

```c
#include <stdio.h>

unsigned int multiply(unsigned int x, unsigned int y)
{
	int i = 0; //代表位数
	while (y)
	{
		if (y&1) x <<= i;
		y >>= 1;
		i++;
	}
	return x;
}

int main()
{
	printf("%d\n", multiply(7, 8));
}
```
 

3、对一个32位无符号整数做循环右移，函数原型是`unsigned int rotate_right(unsigned int x);`。所谓循环右移就是把低位移出去的部分再补到高位上去，例如`rotate_right(0xdeadbeef, 16)`的值应该是0xefdeadbe。

>代码如下：

```c
#include <stdio.h>

unsigned int rotate_right(unsigned int x)
{
	return x >> 1 | x << 31;
};

int main()
{
	printf("%x\n", rotate_right(9));
}
```
 
>值得注意的是，RISC_V提供了循环移位指令`rori a0, a0, 1`。编译时会自动转化为此指令。 
>在数字信号处理中我们也使用过循环移位，之前的取余操作我们也关注过其周期定位能力。

### 其他运算符

条件运算符：`表达式1 ? 表达式2 : 表达式3`，这被广泛用于lambda表达式。

逗号运算符：`表达式1, 表达式2`，非常有趣的运算符，注意参数中填写的并不是逗号运算符。这个运算符基本上是用来蹭表达式1的side effect的。

sizeof运算符：这个运算符是编译时就算好的，运算结果是一个特殊类型：`size_t`，这个`size_t`是一种无符号整形，但具体应该套用哪种类型是implementation defined的，size_t只作为一种别名。

type_def：说起别名，那就是type_def，注意以下代码：

```c
typedef char array_t[10];
array_t a;
```

这里面的a定义出来，其实是 char a[10]的效果。

## 第21章：预处理

预处理始终是一个很神秘的领域，让我们今天来了解一下。

### 预处理的步骤

预处理分为以下几个步骤：

1. 三连符转义，某些终端缺少部分符号，不得不用三连符来进行表示。
2. 将`\`符号续行的语句进行拼接。
3. 将注释，无论是多行注释还是单行注释，替换为一个空格

以上的操作完成后，后面的代码称为“逻辑代码行”。接着逻辑代码行被分为Token和空白字符，此时Token被称为预处理token (preprocessing token)，

4. 针对预处理语句，进行相应的操作。
5. 针对转义序列进行替换，比如将`\n`替换为`0x0a`
6. 拼接相连的字符串
7. 进入流式处理，空白的丢掉，剩下的作为C token，交给编译器。

### 宏定义

函数式宏定义是一种新的宏。变量名宏定义可以使宏像变量一样在代码中使用，而函数式宏定义使你可以像函数一样去调用它。注意这里确实发生了参数的概念，并不是单纯的字母替换。

```c
#define MAX(a, b) ((a)>(b)?(a):(b))
```

在运行时，这部分的作用是将后续代码段中的`#define MAX(a, b)`替换为`(a)>(b)?(a):(b)`这样的条件表达式，其实倒也没什么。

需要注意的点：
1. 宏定义由于是替换，所以括号要尤其注意加上。因为替换之后很有可能运算顺序就错了。
2. 函数式宏定义会导致编译结果的每一处都替换为独立的函数体，因此结果会更大一些
3. 宏定义由于函数参数会被直接替换到函数体内，因此side effect可能会被多次执行。

由于宏定义省去了函数的分配和释放栈、传递参数和传递返回值等过程，因此运算速度会快一些。

### 内联函数

C99加入新关键字inline，用于建立内联函数。内联函数会将函数体直接插入到父作用域代码中，以免去调用的开销。建立的示例如下。

```c
static inline void down_read(struct rw_semaphore *sem)
{
        might_sleep();
        rwsemtrace(sem,"Entering down_read");
        __down_read(sem);
        rwsemtrace(sem,"Leaving down_read");
}
```

这里就是建立了一个内联函数。

### \#、##运算符和可变参数

简单而言，#作为一个特殊标识，会将其后的内容视为字符串

```c
#define STR(s) #s
```

这里的#s并不是引领什么宏语句。整句其实是定义一个宏函数，即 `STR(s)`接受一个参数s，这个s会被替换为字符串。

`##`会将参数作为字符串自动连接起来，以下为用例: 

```c
//预处理之后是`concat`

#define CONCAT(a, b) a##b
CONCAT(con, cat)
```

`#__VA_ARGS__`这个符号用于作为可变参数的替代符号, 用法如下所示

```c
#define showlist(...) printf(#__VA_ARGS__)
```

### 宏展开的过程

当出现多个宏展开时，每次展开都会都头开始遍历所有的宏进行顺序展开。

比如一段宏代码有5个宏替换，那么首先会顺序找到第一个进行替换，然后再从第一个开始顺序找到第一个替换...

### 条件预处理指令

经典的

```c
#ifndef HEADER_FILENAME
#define HEADER_FILENAME
/* body of header */
#endif
```

如果没有定义这个宏，那么就定义这个宏

测试：

```c
#define VERSION  2
#if defined x || y || VERSION < 3
```

尤其注意宏语句的展开顺序：
1. defined 判断x，没有则为0
2. VERSION替换为2
3. y未定义，为0
4. 计算逻辑表达式，为1
5. \#if 1通过

### void类型指针

void类型指针作为通用指针，可以转换成任意类型。其用法如下：

```c
void func(void *pv)
{
	/* *pv = 'A' is illegal */
	char *pchar = pv;
	*pchar = 'A';
}

int main(void)
{
	char c;
	func(&c);
	printf("%c\n", c);
...
}
```

可见这里的void类型指针被传递了char类型的指针。
